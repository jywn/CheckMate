 - 기존의 상용 프로그램들 중 OS 기본탑재 프로그램들(Apple의 Reminders와 Microsoft의 To Do와 같은)은 OS에 종속돼 있어 플랫폼을 오가며 사용하는 데에 불편함이 있었음.

 - 이러한 문제의식을 배경으로 손쉽게 사용할 수 있는 웹 기반 서비스를 만들기로 결정함.

=='''프로젝트 진행'''== 

=== 주요 이슈들 ===
 - 1.Prompt 정확성: ChatGPT를 활용한 정보 parsing이 복잡한 구조를 요구함. 데모는 간소화하여 구현하였으나, 추후 멘토와 협업하여 보완할 예정.
 - 2.Model 설계: 약속의 필수 정보는 '시간'과 '리마인더 여부'로 결정. 기타 정보는 선택적으로 받되 필수 요소는 아님.
 - 3.FBV와 CBV: CRUD 작업은 주로 FBV 사용. 복잡한 로직에는 CBV를 통해 가독성과 유지보수성 향상.
 - 4.코드 양식: Python PEP 8 권장 사항을 따라 주석은 '#'을 사용, 함수와 클래스 설명은 '"""'를 사용하여 구분.
 - 5.프론트엔드 버그 수정: 계획대로 진행 중이며, 주로 마이너 버그 수정. 가능하다면 사이드바 배치, 동적 웹 페이지 등 기능 추가 예정.

=== 개발 중 리스크 ===
 - 1.프라이버시: 개인 정보 보호를 위해 데이터 보안 정책 강화, 정보는 암호화하여 저장, 필요시 익명화 처리.
 - 2.자연어 이해: 오류모델을 다양한 시간 표현 데이터로 학습시켜 인식률 향상, 지속적인 업데이트로 모델 개선.
 - 3.GPT 비용 문제: 토큰 부족 문제 발생시 mBART50과 같은 다른 모델로 대체 고려.

=== 추가 기능 고려 ===
 - 1.Parsing을 통한 Prompt Engineering: 정확한 parsing을 통해 자연스러운 일정 등록 구현
 - 2.STT: 사용자의 음성을 텍스트로 변환해서 입력할 수 있는 기능
 - 3.지도 애플리케이션과의 연동: 카카오맵 API를 통해 일정에서의 장소를 어플에 구현
 - 4.공유 기능: Task들을 카카오톡 혹은 이메일을 통해 지인에게 공유하는 기능 구현

=='''프로젝트 성과'''==

=== 프롬프트 개선사항 ===
 - 개선사항1: 여러 날짜가 입력될 경우, 첫 입력 날짜만 선택하도록 수정함
 - 결과: Output format instruction을 준수하도록 지시
 - 개선사항2: DateTime form을 DB의 form과 일치시킴
 - 결과: db와의 연동성 확보
 - 개선사항3: 시간에 관한 표현이 모호하거나 구어체적인 표현일 경우, 또는 일정이 시간 개념을 내포하고 있는 경우
 - 결과: 문맥으로부터 추론하도록 함
 - 개선사항4: Title에서 Date는 제외한 Location과 People만 출력되도록 설정함
 - 결과: date가 상대적 시간일 경우 유저에게 줄 수 있는 혼란 방지
 - 개선사항5: LLM의 요구 사항에 대한 출력 형태를 제한함
 - 결과: 클라이언트와 통신을 위한 JSON을 반드시 준수

=== 프롬프트 성능검증 ===
 - BLEU Metrics, ROUGE Metrics, Cosine Similarity방식 중 Cosine Similarity 방식을 태책함.
 - BLUE Metrics는 문장의 구성 요소의 순서 변화가 생기면 다른 문장으로 인식
 - ROGUE Metrics는 한국어 문장의 조사와 어미에 변화가 생기면 다른 문장으로 인식
 - 결과: 최종적으로 Cosine Similarity로 결정함.
